Google 機器人驗證 - reCAPTCHA

By Ivan Wu

發表於 2017-04-01

google 提供了打勾驗證機器人的方式，取代輸入驗證碼，大大減少輸入惱人的扭曲驗證碼的方式

近期 google 更是提供了隱形的驗證方式，連打勾都省了，以下範例使用 Laravel 作為示範，實做 google recaptcha

申請：

至 https://www.google.com/recaptcha/admin 做申請

如下圖所示：

  1. 選擇 recaptcha 種類
  2. 設定要使用 recaptcha 的網址，一行一個
  3. 勾選 accept the reCAPTCHA Term of Service
  4. 設定完畢，點選 register

接下來會看到畫面分三個部分

  1. keys：
    Site key：主要放在 HTML 裡，向 google 取得驗證用，是公開的。
    Secret key：表單送出後，由 server 跟 google 驗證是否為有效驗證，勿公開。

  2. client-side integration：
    recaptcha v2 及 invisible 主要差在這個部分而已，以下範例會再詳細說明。

  3. server-side integration：
    與 google 取得驗證用。

實作

在 header 加入 script，取得 google js api

接下來分兩部分實作，將 {YOUR_SITE_KEY} 放入上述提供的 site key

recaptcha v2

invisible recaptcha

button 放入 data-callback="OnSubmitFunction"，在表單送出後會觸發 OnSubmitFunction 這個 function，預設會將 token 帶入 function 第一個參數，以這個例子來講沒有用處，所以不在此討論。

接下來應該就會看到我不是機器人的畫面，
隱形的 recaptcha，應該會在右下角看到 google recaptcha 的 Logo

流程

後端處理

前端會將 token 名稱 g-recaptcha-response 自動 post 到後端，後端將 token 及 secret key POST 到 https://www.google.com/recaptcha/api/siteverify 取得驗證

將 YOUR_SECRET_KEY 替換成你的 secret key

該範例使用 guzzle 處理 post，記得將 composer 加入 guzzle 套件

上述就是一個基本的 google recaptcha 驗證流程，有任何問題歡迎發問！
